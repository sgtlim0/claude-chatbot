## Data Flow Example: Chat Message

1. **HTTP Request** â†’ API Layer (`POST /api/chat`)
2. **Dependency Injection** â†’ Harness gets `SendMessageUseCase` from Container
3. **Use Case Execution** â†’ Application layer orchestrates the flow
4. **Domain Logic** â†’ ChatOrchestrator processes the message
5. **Port Calls** â†’ Domain calls abstract ports (SessionRepository, ChatService)
6. **Adapter Implementation** â†’ Infrastructure adapters handle external calls:
   - MongoSessionRepository saves to MongoDB
   - BedrockChatService streams from AWS Bedrock
7. **Response Stream** â†’ Tokens flow back through layers as SSE

## Key Principles

- **Dependency Direction**: Always inward (outer layers depend on inner)
- **Port Interfaces**: Domain defines interfaces, infrastructure implements
- **Harness Wiring**: Single place to configure all dependencies
- **Test Isolation**: Swap entire infrastructure with TestContainer
- **Domain Purity**: Core logic has zero framework dependencies
